# 04-enterprise-operations

This module focuses on operating the serving stack after deployment.

## File Order

1. `rollout-and-debug.sh`: operational inspection and rollout workflow.
2. `what-to-say-in-interviews.md`: concise interview framing tied to the architecture taught in the repo.

## Why This Exists

Learning how to apply YAML is not enough. Enterprise engineers also need to inspect status, trace generated resources, and explain the platform tradeoffs clearly.
